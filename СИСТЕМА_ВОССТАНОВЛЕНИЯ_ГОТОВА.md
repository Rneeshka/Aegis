# ✅ Система восстановления пароля и исправления

## Что реализовано:

### 1. Система восстановления пароля
- ✅ **Запрос кода**: `/auth/forgot-password` - отправляет код на email
- ✅ **Сброс пароля**: `/auth/reset-password` - меняет пароль по коду
- ✅ **UI в расширении**: форма "Забыли пароль?" с полями для email и кода
- ✅ **Безопасность**: коды истекают через 1 час, одноразовые

### 2. Исправлена проверка истечения API ключей
- ✅ **Строгая проверка**: ключи с истекшим сроком больше не работают
- ✅ **Проверка в middleware**: `validate_api_key()` проверяет `expires_at`
- ✅ **Продление в Admin UI**: истекшие ключи можно продлить

### 3. Упрощена структура базы данных
- ✅ **4 таблицы вместо 6**: accounts, api_keys, threats, logs
- ✅ **Универсальная таблица угроз**: все типы в одной таблице
- ✅ **Компактные API ключи**: убраны избыточные поля
- ✅ **Сохранена функциональность**: все возможности остались

## API Endpoints:

### Восстановление пароля:
```bash
# Запрос кода восстановления
POST /auth/forgot-password
{
  "email": "user@example.com"
}

# Сброс пароля
POST /auth/reset-password
{
  "email": "user@example.com",
  "code": "abc123...",
  "new_password": "newpass123"
}
```

## Как использовать:

### Восстановление пароля:
1. **Забыли пароль?** → Нажмите кнопку "Забыли пароль?"
2. **Введите email** → Укажите email аккаунта
3. **Получите код** → Код придет на email (в логах сервера для тестирования)
4. **Введите код** → Введите код и новый пароль
5. **Готово!** → Пароль изменен, войдите в аккаунт

### Проверка истечения ключей:
- ✅ **Истекшие ключи** больше не работают
- ✅ **Продление** через Admin UI (`http://127.0.0.1:8000/admin/ui`)
- ✅ **Автоматическая проверка** при каждом запросе

## Технические детали:

### База данных:
```sql
-- Новые поля для восстановления
ALTER TABLE accounts ADD COLUMN reset_code TEXT DEFAULT NULL;
ALTER TABLE accounts ADD COLUMN reset_code_expires TIMESTAMP DEFAULT NULL;

-- Упрощенная структура
CREATE TABLE threats (
    type TEXT CHECK(type IN ('url', 'hash', 'ip', 'domain')),
    value TEXT,
    threat_level TEXT CHECK(threat_level IN ('safe', 'suspicious', 'malicious'))
);
```

### Безопасность:
- **Коды восстановления**: случайные 32-символьные hex строки
- **Срок действия**: 1 час с момента генерации
- **Одноразовые**: код удаляется после использования
- **Валидация**: проверка email, длины пароля, формата кода

## Обновление:

1. **Перезапустите сервер**:
   ```bash
   python start_aegis.py
   ```

2. **Обновите расширение**:
   - Удалите старое расширение
   - Загрузите заново из `aegis_start`

3. **Проверьте функциональность**:
   - Регистрация/вход работают
   - Восстановление пароля доступно
   - Истекшие ключи блокируются

## Логи для тестирования:

При запросе восстановления пароля код выводится в логи сервера:
```
Password reset code for user@example.com: a1b2c3d4e5f6...
```

В продакшене этот код будет отправляться на email через SMTP.
