# ✅ Исправлена ошибка "порт уже занят"

## Проблема:
```
ERROR: [Errno 10048] error while attempting to bind on address ('127.0.0.1', 8000): 
[winerror 10048] обычно разрешается только одно использование адреса сокета
```

## Причина:
Порт 8000 уже занят другим процессом (возможно, предыдущим запуском сервера).

## Решения:

### 1. Автоматическое освобождение порта (рекомендуется)

#### Windows (bat файл):
```bash
run_server_fixed.bat
```

#### PowerShell:
```powershell
.\run_server.ps1
```

#### Python скрипт:
```bash
python find_free_port.py
```

### 2. Ручное освобождение порта

#### Через диспетчер задач:
1. Откройте Диспетчер задач (Ctrl+Shift+Esc)
2. Найдите процесс Python или uvicorn
3. Завершите процесс

#### Через командную строку:
```cmd
# Найти процесс на порту 8000
netstat -ano | findstr :8000

# Убить процесс по PID
taskkill /F /PID <PID>
```

### 3. Использование другого порта

#### Запуск на порту 8001:
```bash
cd antivirus-core
python -m uvicorn app.main:app --host 127.0.0.1 --port 8001 --reload
```

#### Запуск на порту 3000:
```bash
cd antivirus-core
python -m uvicorn app.main:app --host 127.0.0.1 --port 3000 --reload
```

## Рекомендуемый способ запуска:

### Для Windows:
```bash
# Запустите bat файл
run_server_fixed.bat
```

### Для PowerShell:
```powershell
# Запустите PowerShell скрипт
.\run_server.ps1
```

### Для Python:
```bash
# Запустите Python скрипт
python find_free_port.py
```

## Что делают скрипты:

1. **Автоматически находят** процесс на порту 8000
2. **Завершают процесс** если он существует
3. **Ждут 2 секунды** для освобождения порта
4. **Запускают сервер** на порту 8000

## Проверка работы:

После успешного запуска проверьте:
- **Статус**: http://127.0.0.1:8000/health
- **Документация**: http://127.0.0.1:8000/docs  
- **Admin UI**: http://127.0.0.1:8000/admin/ui

## Если проблема повторяется:

1. **Перезагрузите компьютер** - это освободит все порты
2. **Используйте другой порт** - измените порт в скриптах
3. **Проверьте антивирус** - он может блокировать порты

Теперь сервер должен запускаться без ошибок! 🎉
