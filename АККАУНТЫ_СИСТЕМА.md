## 📋 Принцип работы

### Базовый доступ (без аккаунта)
- Ключ создается свободным (`user_id = NULL`)
- Базовый функционал доступен без аккаунта
- Анализ по наведению и расширенная аналитика требуют премиум

### Премиум доступ (с аккаунтом)
1. **Создание ключа**: Администратор создает премиум-ключ в Admin UI
2. **Регистрация**: Пользователь создает аккаунт через `/auth/register`
3. **Привязка**: При регистрации автоматически привязывается указанный API ключ
4. **Авторизация**: Пользователь входит через `/auth/login`
5. **Использование**: Теперь ключ привязан к аккаунту и используется

### Важные моменты
- 1 ключ = 1 аккаунт (ключ можно привязать только один раз)
- После привязки ключ больше не доступен другим аккаунтам
- Базовая версия работает без аккаунта
- Премиум требует регистрации и привязки ключа

## 🔌 Примеры использования

### Регистрация аккаунта
```bash
curl -X POST "http://127.0.0.1:8000/auth/register" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "user123",
    "email": "user@example.com",
    "password": "securepass123",
    "api_key": "PREMI-12345-67890-ABCDE-FGHIJ-KLMNO"
  }'
```

Ответ:
```json
{
  "status": "success",
  "message": "Аккаунт успешно создан",
  "user_id": 1,
  "api_key": "PREMI-12345-67890-ABCDE-FGHIJ-KLMNO"
}
```

### Авторизация
```bash
curl -X POST "http://127.0.0.1:8000/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "user123",
    "password": "securepass123"
  }'
```

Ответ:
```json
{
  "status": "success",
  "message": "Успешная авторизация",
  "account": {
    "id": 1,
    "username": "user123",
    "email": "user@example.com",
    "created_at": "2024-01-01T00:00:00",
    "last_login": "2024-01-01T12:00:00"
  },
  "api_keys": [
    {
      "api_key": "PREMI-12345-67890-ABCDE-FGHIJ-KLMNO",
      "name": "Premium Access",
      "access_level": "premium",
      "expires_at": "2025-01-01T00:00:00"
    }
  ]
}
```

### Проверка текущего пользователя
```bash
curl -X GET "http://127.0.0.1:8000/auth/me" \
  -H "X-API-Key: PREMI-12345-67890-ABCDE-FGHIJ-KLMNO"
```

Ответ для привязанного ключа:
```json
{
  "status": "premium",
  "account": {
    "id": 1,
    "username": "user123",
    "email": "user@example.com",
    "created_at": "2024-01-01T00:00:00",
    "last_login": "2024-01-01T12:00:00"
  }
}
```

Ответ для свободного ключа:
```json
{
  "status": "basic",
  "message": "Базовый доступ без аккаунта"
}
```

## 🗄️ Структура базы данных

### Таблица `accounts`
```sql
CREATE TABLE accounts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    email TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login TIMESTAMP DEFAULT NULL,
    is_active BOOLEAN DEFAULT TRUE
);
```

### Таблица `api_keys` (обновлена)
```sql
CREATE TABLE api_keys (
    api_key TEXT PRIMARY KEY,
    ...
    user_id INTEGER DEFAULT NULL,
    FOREIGN KEY (user_id) REFERENCES accounts(id) ON DELETE SET NULL
);
```

## 🎨 Admin UI

В Admin UI теперь отображается колонка "Аккаунт":
- **Свободен** - ключ не привязан к аккаунту (зеленый)
- **username** - ключ привязан к аккаунту с указанным username

## 🔒 Безопасность

- Пароли хешируются с использованием SHA-256 и соли
- Формат хранения: `salt:hash`
- Email и username уникальны
- Минимальная длина пароля: 6 символов
- API ключи можно привязать только один раз
