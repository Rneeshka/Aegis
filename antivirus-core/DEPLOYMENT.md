# 🚀 Руководство по развертыванию Antivirus Core API

## 📋 Предварительные требования

- Python 3.8+
- pip
- Windows PowerShell (для скриптов запуска)

## 🔧 Установка и настройка

### 1. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 2. Настройка переменных окружения
Файл `app/env.env` уже настроен с вашими API ключами:
- ✅ VirusTotal API ключ
- ✅ AbuseIPDB API ключ
- ✅ Безопасный административный токен

### 3. Запуск сервера

#### Локальный запуск:
```bash
python run_local.ps1
```

#### Или напрямую через uvicorn:
```bash
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

## 🔑 Система доступа и API ключи

### Уровни доступа:

#### 🆓 **Базовый доступ (без ключа)**
- ✅ Проверка URL (локальный анализ)
- ✅ Проверка файлов по хэшу (локальная база)
- ✅ Проверка доменов (локальная база)
- ❌ Внешние API недоступны
- ❌ Анализ по наведению недоступен
- ❌ Проверка IP адресов недоступна

#### 💎 **Премиум доступ (с API ключом)**
- ✅ Все функции базового доступа
- ✅ Расширенный анализ через внешние API (VirusTotal, AbuseIPDB)
- ✅ Проверка IP адресов
- ✅ Анализ по наведению (hover analysis)
- ✅ Повышенные лимиты запросов

### Формат API ключей:
```
BASIC-XXXXX-XXXXX-XXXXX-XXXXX-XXXXX  (базовый уровень)
PREMI-XXXXX-XXXXX-XXXXX-XXXXX-XXXXX  (премиум уровень)
```

### Создание API ключа через админ панель:
1. Откройте http://127.0.0.1:8000/admin/ui
2. Перейдите в раздел "Ключи API"
3. Выберите уровень доступа (базовый/премиум)
4. Установите лимиты запросов
5. Создайте ключ

### Использование API:

#### Без ключа (базовый доступ):
```bash
# Проверка URL
curl -X POST "http://127.0.0.1:8000/check/url" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com"}'
```

#### С премиум ключом:
```bash
# Проверка URL с расширенным анализом
curl -X POST "http://127.0.0.1:8000/check/url" \
  -H "X-API-Key: PREMI-XXXXX-XXXXX-XXXXX-XXXXX-XXXXX" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com"}'

# Проверка IP адреса (только премиум)
curl -X GET "http://127.0.0.1:8000/check/ip/8.8.8.8" \
  -H "X-API-Key: PREMI-XXXXX-XXXXX-XXXXX-XXXXX-XXXXX"

# Анализ по наведению (только премиум)
curl -X POST "http://127.0.0.1:8000/check/hover" \
  -H "X-API-Key: PREMI-XXXXX-XXXXX-XXXXX-XXXXX-XXXXX" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com", "domain": "example.com"}'
```

## 📊 Мониторинг и логи

```bash
curl http://127.0.0.1:8000/health
```

### Логи:

- Автоматическая ротация каждые 7 дней
- Максимальный размер файла: 100MB

### Админ панель:
- URL: http://127.0.0.1:8000/admin/ui
- Статистика использования API
- Управление ключами
- Просмотр логов

## 🐛 Устранение неполадок

### Проблемы с внешними API:
- Проверьте правильность API ключей в `app/env.env`
- Убедитесь в наличии интернет-соединения
- Проверьте лимиты API (VirusTotal: 500/час)

### Проблемы с базой данных:
- Убедитесь, что папка `data/` существует
- Проверьте права доступа к файлам БД
- При необходимости удалите файлы БД для пересоздания

### Проблемы с логами:
- Проверьте права доступа к папке `logs/`
- Убедитесь в наличии свободного места на диске

## 📈 Производительность

### Настройки по умолчанию:
- Максимальный размер файла: 50MB
- Максимальная длина URL: 2048 символов
- Кэширование результатов: 5 минут
- Rate limiting: настраивается по ключам

### Оптимизация:
- Используйте SSD для базы данных
- Настройте Redis для кэширования
- Увеличьте лимиты памяти для больших файлов

## 🔄 Обновления

При обновлении:
1. Создайте резервную копию базы данных
2. Обновите зависимости: `pip install -r requirements.txt --upgrade`
3. Перезапустите сервер
4. Проверьте работоспособность через `/health`

## 📞 Поддержка

При возникновении проблем:
1. Проверьте логи в папке `logs/`
2. Используйте `/health` для диагностики
3. Проверьте конфигурацию в `app/env.env`
