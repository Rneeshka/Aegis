# 🔧 Диагностика проблемы с расширением

## Проблема
- Сервер запущен
- Расширение не подключается
- Настройки не открываются

## Решение

### 1. Перезагрузите сервер
```bash
# Остановите старый сервер (Ctrl+C)
# Запустите заново
python start_aegis.py
```

### 2. Перезагрузите расширение в браузере

#### Chrome/Edge:
1. Откройте `chrome://extensions/`
2. Найдите "SafeBrowse"
3. Нажмите **🔄 Перезагрузить** (кнопка обновления)
4. Или выключите/включите расширение

### 3. Откройте настройки

**Способ 1: Через всплывающее окно**
- Кликните на иконку расширения в панели инструментов
- Нажмите "Настройки"

**Способ 2: Через меню расширений**
- Правый клик на иконке расширения
- Выберите "Параметры расширения" или "Options"

**Способ 3: Прямая ссылка**
- Откройте `chrome://extensions/`
- Найдите "SafeBrowse"
- Нажмите "Подробнее" → "Параметры расширения"

### 4. Проверьте консоль браузера

1. Откройте DevTools (F12)
2. Перейдите в **Console**
3. Откройте настройки расширения
4. Проверьте на наличие ошибок

### 5. Проверьте service worker

1. Откройте `chrome://extensions/`
2. Найдите "SafeBrowse"
3. Нажмите "Просмотреть служебный работник" (Inspect service worker)
4. Проверьте на наличие ошибок в Console

### 6. Проверьте права доступа

Убедитесь, что расширение имеет доступ к:
- ✅ Storage (для сохранения настроек)
- ✅ Host permissions (для доступа к API)
- ✅ Scripting (для content scripts)

### 7. Тест сервера

Откройте в браузере:
- http://127.0.0.1:8000/health
- http://127.0.0.1:8000/

Должен вернуться JSON с состоянием сервера.

### 8. Проверьте API Base URL в настройках

При открытии настроек:
1. Прокрутите до "API Base URL"
2. Должно быть: `http://127.0.0.1:8000`
3. Если нет - введите вручную

## Типичные проблемы

### Ошибка: "Service worker registration failed"
- **Решение**: Перезагрузите расширение (кнопка 🔄)

### Ошибка: "Failed to fetch"
- **Решение**: Проверьте, что сервер запущен на `http://127.0.0.1:8000`

### Настройки не открываются
- **Решение**: Проверьте `options.html` в папке `aegis_start`

### Расширение не работает
- **Решение**: Проверьте `manifest.json` на наличие всех permissions

## Быстрая проверка

```bash
# 1. Проверьте, работает ли сервер
curl http://127.0.0.1:8000/health

# 2. Должен вернуться JSON с "status": "success"
```

## Если ничего не помогло

1. Переустановите расширение:
   - Удалите из `chrome://extensions/`
   - Загрузите заново из папки `aegis_start`

2. Проверьте логи сервера:
   - Запустите сервер в отдельном окне
   - Посмотрите на наличие ошибок

3. Проверьте Browser Console:
   - F12 → Console
   - Ищите ошибки с "chrome-extension://"
